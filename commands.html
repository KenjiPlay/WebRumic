ruleta / roulette / r

Juega una ruleta simple apostando a color, paridad o número exacto.

Estructura !r <apuesta> <rojo|negro|par|impar|0-36>
Ejemplo: rojo !r 100 rojo
Ejemplo: negro !r 100 negro
Ejemplo: par !r 100 par
Ejemplo: impar !r 100 impar
Ejemplo: número exacto !r 100 17
Opciones disponibles:
  • rojo o red — apuesta a que sale rojo.
  • negro o black — apuesta a que sale negro.
  • par o even — apuesta a número par.
  • impar o odd — apuesta a número impar.
  • 0-36 — apuesta a un número exacto.
Pagos:
  • rojo, negro, par e impar pagan x2.
  • Número exacto paga x36.
Reglas:
  • La ruleta tira un número entre 0 y 36.
  • El 0 es verde.
  • Si sale 0, las apuestas a rojo, negro, par e impar pierden.
  • Para ganar con número exacto, debe salir exactamente el número elegido.

🛒 Admin: Tienda

addshopitem

Crea un item nuevo en la tienda del servidor.

Estructura !addshopitem <id> <precio> <categoria> <emoji|-> "nombre" "descripcion"
Ejemplo !addshopitem espada 500 armas ⚔️ "Espada básica" "Una espada simple para principiantes."
Ejemplo sin emoji !addshopitem ticket 100 tickets - "Ticket común" "Un ticket para eventos."
Validaciones:
  • id solo acepta letras, números, guiones y guiones bajos. Máximo 40 caracteres.
  • precio debe estar entre 1 y 1.000.000.000.
  • categoria no puede superar 32 caracteres.
  • nombre no puede superar 80 caracteres.
  • descripcion no puede superar 500 caracteres.
  • Usa - en emoji si quieres crear el item sin emoji.

delshopitem

Elimina un item de la tienda junto con sus datos asociados.

Estructura !delshopitem <id>
Ejemplo !delshopitem espada
También elimina:
  • El item de los inventarios de usuarios.
  • Los requisitos del item.
  • Las acciones configuradas del item.
  • El registro del item en la tienda.

editshopitem

Edita un campo específico de un item existente.

Estructura !editshopitem <id> <campo> <valor>
Ejemplo: precio !editshopitem espada price 750
Ejemplo: nombre !editshopitem espada name "Espada legendaria"
Ejemplo: quitar emoji !editshopitem espada emoji off
Campos editables:
  • name — cambia el nombre. Máximo 80 caracteres.
  • price — cambia el precio. Entre 1 y 1.000.000.000.
  • category — cambia la categoría. Máximo 32 caracteres.
  • emoji — cambia el emoji. Usa -, off o none para quitarlo.
  • description — cambia la descripción. Máximo 500 caracteres.

addreq

Agrega un requisito a un item de la tienda. El usuario deberá cumplir ese requisito para poder comprar el item.

Estructura !addreq <itemId> <role|item> <valor>
Ejemplo: requisito de rol !addreq espada role @VIP
Ejemplo: requisito de item !addreq espada item licencia
Tipos de requisito permitidos:
  • role — el usuario necesita tener un rol específico. Puedes usar una mención de rol o su ID.
  • item — el usuario necesita tener un item existente en la tienda.
Notas:
  • El itemId debe existir en la tienda.
  • Si usas item, el item requerido también debe existir en la tienda.
  • Este comando no consume el item requerido; solo valida que el usuario lo tenga.

addaction

Agrega una acción a un item. Las acciones se ejecutan cuando el usuario usa el item con !use <item>.

Estructura !addaction <itemId> "nombreAccion" <tipo> <valor>
Ejemplo: dar rol !addaction vip_ticket "dar vip" giverol @VIP
Ejemplo: quitar rol !addaction pase_castigo "quitar vip" delrol @VIP
Ejemplo: alternar rol !addaction toggle_vip "alternar vip" togglerol @VIP
Ejemplo: dar dinero !addaction bolsa "premio" givemoney 500
Ejemplo: quitar dinero !addaction multa "cobrar multa" delmoney 250
Ejemplo: dar item !addaction caja "dar espada" give_item espada 2
Ejemplo: quitar item !addaction llave "consumir llave" remove_item llave 1
Ejemplo: enviar mensaje !addaction pase "mensaje" sendmessage "Hola {user}, ahora tienes {tagrol}"
Tipos de acción de dinero:
  • givemoney — da monedas al usuario.
  • delmoney — quita monedas al usuario.
  • give_coins — alias para dar monedas.
  • remove_coins — alias para quitar monedas.
Tipos de acción de rol:
  • giverol — da un rol al usuario.
  • delrol — quita un rol al usuario.
  • togglerol — si el usuario tiene el rol, se lo quita; si no lo tiene, se lo da.
  • give_role — alias para dar rol.
  • remove_role — alias para quitar rol.
Tipos de acción de item:
  • give_item — da un item al usuario. Usa <itemId> [cantidad].
  • remove_item — quita un item al usuario. Usa <itemId> [cantidad].
Tipos de acción de texto:
  • sendmessage — envía un mensaje generado desde el valor configurado.
Límites y validaciones:
  • El itemId debe existir en la tienda.
  • El nombre de la acción no puede superar 50 caracteres.
  • No pueden existir dos acciones con el mismo nombre en el mismo item.
  • Las cantidades máximas para dinero o items son de 1.000.000.000.
  • Los mensajes de sendmessage no pueden superar 1000 caracteres.

addmoney

Añade dinero a la wallet de un usuario o a todos los miembros de un rol.

Estructura: usuario !addmoney @usuario <cantidad>
Estructura: rol !addmoney @rol <cantidad>
Ejemplo: usuario !addmoney @Neko 1000
Ejemplo: rol !addmoney @Miembros 500
Funcionamiento:
  • Si mencionas un usuario, añade dinero solo a ese usuario.
  • Si mencionas un rol y no un usuario, añade dinero a todos los miembros no bot de ese rol.
  • El dinero se añade a la wallet, no al banco.
  • La cantidad máxima permitida es 1.000.000.000.
Notas sobre roles:
  • El bot intenta cargar los miembros del servidor antes de aplicar el dinero.
  • Los bots no reciben dinero.
  • El resultado muestra usuarios afectados y fallos si los hubo.

delmoney

Quita dinero de la wallet de un usuario o de todos los miembros de un rol con saldo suficiente.

Estructura: usuario !delmoney @usuario <cantidad>
Estructura: rol !delmoney @rol <cantidad>
Ejemplo: usuario !delmoney @Neko 500
Ejemplo: rol !delmoney @Miembros 250
Funcionamiento:
  • Si mencionas un usuario, quita dinero solo a ese usuario.
  • Si mencionas un rol y no un usuario, intenta quitar dinero a todos los miembros no bot de ese rol.
  • El dinero se quita de la wallet, no del banco.
  • La cantidad máxima permitida es 1.000.000.000.
Notas sobre roles:
  • Solo se afecta a usuarios con saldo suficiente en wallet.
  • Los usuarios sin dinero suficiente se cuentan como omitidos.
  • Los bots no son afectados.
  • El resultado muestra usuarios afectados, omitidos y fallos si los hubo.

🎰 Admin: Juegos

addslot

Añade o actualiza un símbolo de slots para el servidor.

Estructura !addslot <emoji> <rareza> <multiplicador> <weight> <maxPorColumna>
Ejemplo !addslot 🍒 comun 2 20 2
Ejemplo con otro símbolo !addslot 💎 legendario 10 2 1
Parámetros:
  • emoji — símbolo que aparecerá en la tragamonedas.
  • rareza — texto libre guardado en minúsculas. Ejemplos: comun, raro, epico, legendario.
  • multiplicador — cuánto multiplica la apuesta cuando el símbolo paga.
  • weight — peso/probabilidad relativa de aparición del símbolo.
  • maxPorColumna — máximo de veces que ese símbolo puede aparecer por columna.
Límites:
  • multiplicador debe ser mayor que 0 y máximo 1000.
  • weight debe ser un entero entre 1 y 10000.
  • maxPorColumna debe estar entre 1 y 3.
Cómo interpretar weight:
  • Un símbolo con weight alto aparece más seguido.
  • Un símbolo con weight bajo aparece menos seguido.
  • Ejemplo: un símbolo con weight 40 será mucho más común que uno con weight 2.
Notas:
  • Si agregas un emoji que ya existe, se actualiza su rareza, multiplicador, weight y máximo por columna.
  • La rareza no tiene una lista cerrada en este archivo; puedes usar nombres personalizados.
  • El orden del listado interno prioriza weight, multiplicador y emoji según la base de datos.

setcrateimage

Configura o elimina la imagen de una crate.

Estructura: configurar !setcrateimage <crateId> <url>

addcratereward

Añade una recompensa posible dentro de una crate.

Estructura !addcratereward <crateId> <rewardName> <weight>
Ejemplo !addcratereward crate_rara monedas 50
Parámetros:
  • crateId — ID de la crate donde se agregará la recompensa.
  • rewardName — nombre interno de la recompensa.
  • weight — probabilidad relativa de que salga esa recompensa.
Weight:
  • Debe ser un número entero mayor que 0.
  • Una recompensa con más weight aparece más seguido.
  • Ejemplo: una recompensa con weight 50 es mucho más común que una con weight 5.
Notas:
  • La crate debe existir.
  • No puedes crear dos recompensas con el mismo nombre dentro de la misma crate.
  • Crear la recompensa no entrega nada todavía; debes agregarle acciones con addcrateaction.

delcratereward

Elimina una recompensa de una crate.

Estructura !delcratereward <crateId> <rewardName>
Ejemplo !delcratereward crate_rara monedas
Notas:
  • La recompensa debe existir dentro de esa crate.
  • Al eliminar la recompensa, deja de poder salir al abrir la crate.

addcrateaction

Añade una acción a una recompensa de crate. Las acciones se ejecutan cuando esa recompensa sale al abrir la crate.

Estructura !addcrateaction <crateId> <rewardName> <tipo> [valor] [cantidad]
Ejemplo: dar dinero !addcrateaction crate_rara monedas givemoney 1000
Ejemplo: dar item !addcrateaction crate_rara espada give_item espada 1
Ejemplo: dar rol !addcrateaction crate_rara vip giverol 123456789012345678
Ejemplo: enviar mensaje !addcrateaction crate_rara aviso sendmessage "Felicidades {user}, ganaste una recompensa."
Tipos permitidos:
  • givemoney — entrega monedas al usuario.
  • give_item — entrega un item al usuario.
  • giverol — entrega un rol al usuario.
  • sendmessage — envía un mensaje configurado.
Formato por tipo:
  • givemoney: !addcrateaction <crateId> <rewardName> givemoney <cantidad>
  • give_item: !addcrateaction <crateId> <rewardName> give_item <itemId> <cantidad>
  • giverol: !addcrateaction <crateId> <rewardName> giverol <roleId>
  • sendmessage: !addcrateaction <crateId> <rewardName> sendmessage <mensaje>
Validaciones:
  • La crate debe existir.
  • La recompensa debe existir dentro de esa crate.
  • givemoney requiere una cantidad entera mayor que 0.
  • give_item requiere itemId y cantidad entera mayor que 0.
  • giverol requiere un ID de rol.
  • sendmessage requiere un mensaje no vacío.
Notas:
  • En sendmessage, puedes usar comillas si el mensaje tiene espacios.
  • Este archivo solo acepta roleId para giverol, no mención de rol.
  • Si quieres usar variables como {user}, dependen de cómo esté implementado el renderizado al usar la crate.

📜 Admin: Misiones

addmission

Crea una misión nueva con recompensa configurable.

Estructura !addmission <id> "titulo" "descripcion" <missionType> <rewardType> [rewardValue] [rewardAmount]
Ejemplo: dinero !addmission diaria "Misión diaria" "Completa tu misión diaria." daily givemoney - 500
Ejemplo: item !addmission espada "Misión de espada" "Reclama una espada especial." manual item espada 1
Ejemplo: mensaje !addmission aviso "Misión de aviso" "Recibe un mensaje especial." manual message "Felicidades {user}, completaste {mission}."
missionType:
  • missionType se guarda como texto en minúsculas.
  • No tiene una lista cerrada en este archivo, así que puedes usar tipos personalizados como daily, manual, evento, semanal, etc.
rewardType permitidos:
  • givemoney — entrega monedas. Requiere rewardAmount mayor que 0.
  • item — entrega un item. Requiere rewardValue como itemId y rewardAmount mayor que 0.
  • message — envía un mensaje en el canal. Requiere rewardValue con el texto del mensaje.
Variables para rewardType message:
  • {user} — menciona al usuario que completó la misión.
  • {username} — nombre del usuario.
  • {mission} — título de la misión.
  • {missionid} — ID de la misión.

editmission

Edita un campo específico de una misión existente.

Estructura !editmission <id> <campo> <valor>
Ejemplo: título !editmission diaria title "Nueva misión diaria"
Ejemplo: recompensa !editmission diaria reward_type givemoney
Ejemplo: cantidad !editmission diaria reward_amount 1000
Ejemplo: activar/desactivar !editmission diaria is_active off
Campos editables:
  • title — cambia el título de la misión.
  • description — cambia la descripción.
  • mission_type — cambia el tipo de misión. Se guarda en minúsculas.
  • reward_type — cambia el tipo de recompensa. Solo acepta givemoney, item o message.
  • reward_value — cambia el valor de la recompensa. Para item es el itemId; para message es el texto.
  • reward_amount — cambia la cantidad. Debe ser un número entero mayor o igual que 0.
  • is_active — activa o desactiva la misión.
Valores permitidos para is_active:
  • 1, on, true — activa la misión.
  • 0, off, false — desactiva la misión.
